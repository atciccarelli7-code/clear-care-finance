# Blood Thinner Discharge Readiness pilot measurement plan

## Evaluation question

Can a structured staff-guided workflow improve demonstrated understanding and surface discharge barriers without adding unacceptable bedside burden?

## Design

Start with one unit, one discharge setting, approved medicine branches, and synthetic workflow simulation before limited patient use. Compare baseline discharge practice with the pilot using the same operational definitions. Do not claim reduced readmissions or adverse events without an adequately designed evaluation.

## Measures

| Domain | Measure | Definition |
| --- | --- | --- |
| Adoption | Eligible workflow start rate | Starts / eligible discharge opportunities |
| Fidelity | Exact branch match rate | Correct medicine and regimen branch / started workflows |
| Comprehension | First-pass teach-back rate | Tasks passed without prompting / attempted tasks |
| Recovery | Re-teach recovery rate | Tasks passed after re-teaching / tasks initially not passed |
| Safety | Stop rate | Workflows blocked by failed teach-back or unresolved barrier / starts |
| Continuity | Barrier identification rate | Workflows with at least one barrier / starts |
| Resolution | Safe resolution rate | Barriers resolved or open with a safe backup / identified barriers |
| Workflow | Median active staff time | Start-to-handoff active minutes, excluding interruptions when measurable |
| Experience | Patient usefulness | Fixed response collected in approved non-identifying survey |
| Equity | Interpreter/accessibility completion | Required access support completed / workflows needing support |

## Data minimization

Website analytics are consent-gated and limited to fixed workflow, stage, mode, status, and aggregate count fields. No patient-level medication choice, teach-back answer, barrier type, clinical value, identifier, or free text is sent. Institutional pilot data requires a separate approved privacy, security, retention, access, and analysis plan.

## Stop rules

Pause the pilot for a suspected content-related safety event, wrong medication branch, printing without required product binding, missing emergency/contact localization, repeated serious teach-back failure, inaccessible critical content, or inability to identify distributed versions.
