# Medicare and Medicaid Guide PDF Layout and Visual Refinement Report

Community Acquired Finance  
PDF layout and visual refinement pass  
Last updated: 2026-07-07

## Scope

This pass refines the pre-public Medicare/Medicaid guide PDF builder before the final public PDF binary is committed.

The goal is to reduce awkward open space, preserve readability, and add a small number of elegant explanatory visuals that make the guide easier for stressed families and caregivers to understand.

This pass does not publish the final public PDF. It does not add any PDF under `/public/guides/` or `/public/drafts/`. It does not unlock the landing page CTA, add the PDF URL to the sitemap, generate QR codes, add ads, add affiliate links, add insurer rankings, add plan recommendations, add lead forms, or add sales language.

## Files changed

Expected changed files:

- `/scripts/build-medicare-medicaid-guide-pdf.mjs`
- `/docs/medicare-medicaid-guide-pdf-layout-visual-refinement-report.md`

## Layout and spacing changes

The PDF builder styling was tightened without changing the source manuscript text.

Changes made:

- Reduced PDF page margins from `0.65in 0.7in` to `0.6in 0.66in`.
- Slightly reduced body size/leading from `11pt/1.5` to `10.8pt/1.46`.
- Reduced top/bottom spacing around headings.
- Reduced paragraph, list, blockquote, callout, card, and footer spacing.
- Reduced worksheet row height and padding while preserving writing room.
- Reduced QR placeholder box size slightly.
- Relaxed `keep-soft` page-break rules so callouts, examples, tools, and source notes can flow more naturally instead of pushing large blocks to the next page.
- Kept the direct-answer box protected from splitting because it functions as the chapter summary anchor.
- Preserved chapter start behavior so each chapter still begins cleanly.

## Visuals added

The visuals are native HTML/CSS blocks generated by the PDF builder. No image assets, clip art, stock illustrations, or decorative graphics were added.

### 1. Guide-use map

Placement: `How to use this guide` page.

Purpose: Helps a reader move from the problem in front of them to the verification step: discharge, coverage, documents, cost, and source verification.

Rationale: This gives the guide a simple mental model without adding decorative content.

### 2. Medicare vs Medicaid comparison visual

Placement: Chapter 4, `Medicare vs Medicaid`.

Purpose: Shows Medicare and Medicaid side by side as different programs that answer different questions.

Rationale: This is the foundational distinction families need before they can understand rehab, long-term care, Medicaid, dual eligibility, and bills.

### 3. Bill review comparison visual

Placement: Chapter 8, `Why Medicare Can Pay and You Still Owe`.

Purpose: Shows that a provider bill should be compared against the Medicare Summary Notice or Explanation of Benefits before payment decisions.

Rationale: This turns a confusing billing concept into a practical review process and directly supports the EOB-to-bill checker tool.

### 4. Discharge pathway visual

Placement: Chapter 10, `What to Ask Before Discharge`.

Purpose: Shows the main discharge paths: home health, SNF/rehab, and long-term care, while emphasizing payer verification.

Rationale: This is one of the highest-stress moments for families. A pathway visual helps the reader see that care setting and payer rule are separate but connected questions.

### 5. SNF timeline visual

Placement: Chapter 12, `Skilled Nursing Facility Care`.

Purpose: Shows SNF coverage as short-term and rule-based: qualifying rule, early stay, ongoing review, and the change in question after skilled care ends.

Rationale: Families often confuse short-term skilled facility coverage with long-term custodial care. The timeline supports that distinction without making coverage promises.

## Visual style guardrails

The visual system uses:

- restrained border/card layouts,
- muted accent color already used by the guide,
- short labels and compact explanatory text,
- no cartoon/clip-art elements,
- no decorative images,
- no final QR codes,
- print-friendly native HTML/CSS.
